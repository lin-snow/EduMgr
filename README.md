# EduMgr——数据库课设



## 一、系统概述

本系统为 **教学管理系统**，主要实现对学生、教职工、系、课程及成绩信息的管理。系统采用 **B/S 架构**，前端使用 **Nuxt.js** 框架，后端采用 **Golang + Echo** 框架，数据库使用 **PostgreSQL**，数据库访问使用 **GORM**，数据库初始化和版本管理使用 **golang-migrate**。

系统功能包括：

1. 学生信息管理（增删改查、转学、毕业、历史数据存储）。
2. 教职工信息管理。
3. 系信息管理。
4. 课程管理与选课管理。
5. 成绩录入与查询。
6. 查询统计与报表输出。
7. 用户权限与安全控制。

## 二、技术选型

| 组件       | 技术                             | 说明                                               |
| ---------- | -------------------------------- | -------------------------------------------------- |
| 前端       | Next.js && Shadcn UI+TailwindCSS | 丰富的组件生态与方便的UI设计                       |
| 后端       | Golang + Echo                    | 高性能 Web 框架，支持中间件、路由和 REST API。     |
| ORM        | GORM                             | Golang ORM，用于对象关系映射与数据库操作。         |
| 数据库迁移 | golang-migrate                   | 管理数据库 SQL 文件版本和迁移，数据库初始化。                    |
| 数据库     | PostgreSQL                       | 开源关系型数据库，支持事务、索引、完整性约束。     |
| 认证授权   | JWT / Role-based                 | 基于角色的权限控制，分配学生、教师、管理员等权限。 |

均使用latest version



## 三、需求分析

### 1. 功能需求

#### 学生信息管理

- 新增、修改、删除学生信息。
- 按学号查询学生信息。
- 转学、毕业学生数据归档到历史库。

#### 教职工信息管理

- 新增、修改、删除教职工信息。
- 按职工号或姓名查询信息。

#### 系信息管理

- 新增、修改、删除系信息。
- 查询系信息。

#### 课程管理

- 新增、修改、删除课程信息。
- 查询课程信息。
- 课程与教师绑定。
- 选课管理：学生每学期选课总学分 ≤ 15。

#### 成绩管理

- 录入与修改成绩（平时、考试、总评）。
- 按课程、学生查询成绩。
- 生成成绩报表。

#### 报表统计

- 成绩登记表、成绩报表。
- 按课程、教师、系号分组统计。
- 输出分数区间统计（≥90、≥80、≥70、≥60、不及格）。
